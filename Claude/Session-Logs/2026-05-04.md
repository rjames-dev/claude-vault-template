## Key Findings

**socratics-v2-prototype:** Self-contained MVP of "Socratics Compose" (XLSX financial consolidation via AI). Flask backend (port 5050) + Next.js 15 frontend (port 3000). Uses claude-haiku-4-5-20251001. State is JSON files on disk (ephemeral). Deployed on Azure Container Apps + Vercel. No dependency on the other two repos.

**v2-backend:** Production replacement for the prototype's Flask backend. Phase 0 landing zone — three services (API, Compute Worker, Ingest Worker) and five packages (llm_wrapper, queue_client, document_client, rate_limit, multidoc). Exposes identical 17 endpoints to the prototype. Kind handlers wired but stubbed. Key invariant: llm_wrapper is the sole LLM import point, enforced by CI lint.

**v2-frontend (Agora):** Production frontend. Next.js 15, Auth0, Hasura GraphQL, Redux + IndexedDB. BFF pattern — all backend calls proxied through Next.js API routes. "Logos backend" (the Flask API) consumed via `/api/logos/*` routes. AI chat via SSE Observable stream.

**Critical interconnection:** Both prototype and v2-backend/services/api expose the same 17 endpoints. v2-frontend proxies to whichever is configured as the Logos backend. Migration is an internal swap invisible to the frontend.

**Named mismatch:** Frontend uses `IS`/`BS`; backends use `pl`/`balance_sheet`. Translated at BFF layer.
